# Contributing to ContextOS

- Read `AGENTS.md` before making changes.
- Work one approved milestone at a time.
- Keep changes small, scoped, and aligned with the active milestone spec.
- Use a topic branch and write clear commit messages when you are ready to commit.
- Add or update focused tests and documentation when behavior changes.
- Do not introduce secrets, fake metrics, or deployment activity without approval.
- Do not add runtime dependencies unless the milestone explicitly requires them.

Documentation and validation matter here as much as code quality. If a change affects scope, architecture, security, or operations, update the relevant docs in the same milestone.
